# BUG-008 — 도면 기능 다수 수정사항

## 수정 범위
- `src/pages/LayoutEditorPage.tsx`
- `src/components/layout-editor/EditorCanvas.tsx`
- `src/components/layout-editor/EditorToolbar.tsx`
- `src/components/layout-editor/LayerPanel.tsx`
- `src/stores/layoutEditorStore.ts`

---

## 수정 1 — 줌 인/아웃 기준점 변경

**현재 상태:**
- 마우스 스크롤 줌 인/아웃 시 좌측 상단을 기준으로 확대/축소됨

**변경 요구사항:**
- 마우스 커서 위치를 기준으로 줌 인/아웃 되도록 변경

---

## 수정 2 — 도면 모드 구조 변경 및 명칭 수정

**현재 상태:**
- 보기 / 편집 / 초기설정 3가지 모드로 구성

**변경 요구사항:**
- 편집 모드 완전 제거
- 초기설정 모드 명칭을 `편집`으로 변경
- 최종 구조: 보기 / 편집(기존 초기설정) 2가지 모드만 유지
- 편집 모드 제거로 인해 연결된 버튼/메뉴 항목도 함께 정리

---

## 수정 3 — 초기 레이아웃 배경 영역 설정 방식 변경

**현재 상태:**
- '배경 이미지를 업로드 하세요' 라고 쓰인 흰색 박스가
  기본 영역으로 표시됨

**변경 요구사항:**
- 흰색 기본 박스 제거
- 아래 두 가지 방식으로 기본 영역 설정:

  **방식 A — 배경 이미지 업로드**
  - 사용자가 업로드한 이미지가 도면의 바탕이 됨
  - 이미지 크기가 곧 기본 영역 크기

  **방식 B — 이미지 없음 (직접 크기 설정)**
  - `이미지 없음` 라디오 버튼 선택 시
  - 가로(px) / 세로(px) 입력 필드 표시
  - 입력한 사이즈로 기본 영역 생성

---

## 수정 4 — 레이아웃 화면 열 때 전체 영역 화면에 맞게 표시

**현재 상태:**
- 레이아웃 화면 진입 시 도면이 화면에 맞게 자동 조정되지 않음

**변경 요구사항:**
- 레이아웃 화면 최초 진입 시
  수정 3에서 설정한 기본 영역 전체가 화면에 꽉 차도록
  자동 줌 레벨 및 위치 조정

---

## 수정 5 — 도면 이동 범위 제한

**현재 상태:**
- 마우스 드래그로 도면 이동 시 기본 영역을 벗어날 수 있음

**변경 요구사항:**
- 마우스 드래그 이동 시 수정 3에서 설정한
  기본 영역(이미지 크기 또는 설정한 px 사이즈)을
  상하좌우로 벗어나지 않도록 이동 범위 제한

---

## 수정 6 — 거리 표시 축척 설정 후 활성화

**현재 상태:**
- 도면 좌측 하단의 거리 표시가 축척 설정 전부터 표시됨

**변경 요구사항:**
- 축척 설정(픽셀-거리 입력 3회) 완료 전까지 거리 표시 비활성화
- 축척 설정 완료 후 해당 축척 기준으로 거리 표시 활성화

---

## 수정 7 — 설정 수정 취소 시 변경사항 미저장 및 저장 버튼 활성화

**현재 상태:**
- 편집(초기설정) 모드에서 설정 수정 취소 클릭 시
  수정 중이던 내용이 저장되는 오류 발생
- 저장 버튼이 비활성화된 것처럼 회색으로 표시됨

**변경 요구사항:**
- 설정 수정 취소 클릭 시 수정 중이던 내용 전체 취소
  (저장 전 상태로 복원)
- 저장 버튼 활성화 상태로 표시 (회색 → 정상 색상)

---

## 수정 8 — 설비 레이어 Y축 방향 반전

**현재 상태:**
- Y축 기준: 좌측 상단이 0, 하단으로 내려갈수록 수치 증가

**변경 요구사항:**
- Y축 기준 반전: 좌측 하단이 0, 상단으로 올라갈수록 수치 증가
- 기존에 입력된 설비 좌표 데이터도 반전 기준으로 재계산하여 렌더링
