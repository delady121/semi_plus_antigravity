# FEAT-001 — 레이아웃 기능 수정사항

## 수정 범위
- `src/pages/LayoutEditorPage.tsx`
- `src/components/layout-editor/EditorCanvas.tsx`
- `src/components/layout/GNB.tsx` 또는 `LNB.tsx` (네비게이션 바 관련)

---

## 수정 항목 1 — 썸네일 카드 UI 변경

**현재 상태:**
- 각 레이아웃 썸네일 카드에 `보기` 버튼과 `편집` 버튼이 별도 존재

**변경 요구사항:**
- `보기` 버튼, `편집` 버튼 모두 제거
- 썸네일 카드 어느 곳을 클릭해도 기존 `보기` 버튼과 동일한 기능으로 진입
- 단, 레이아웃 초기설정이 완료되지 않은 상태의 카드에 있는 `설정 계속` 버튼은 유지

---

## 수정 항목 2 — 네비게이션 바 편집 기능 변경

**현재 상태:**
- 네비게이션 바에 연필 아이콘(편집 기능) 존재
- 레이아웃 메뉴에 `새 레이아웃` 버튼만 존재

**변경 요구사항:**
- 네비게이션 바의 연필 아이콘 편집 기능 제거
- `새 레이아웃` 버튼은 유지
- `새 레이아웃` 버튼 옆에 `편집` 버튼 추가
- `편집` 버튼 기능:
  - 개별 레이아웃 명칭 변경
  - 개별 레이아웃 삭제

---

## 수정 항목 3 — 레이아웃 에디터 캔버스 렌더링 문제

**현재 상태:**
- 목업 데이터의 설비 및 OHT 레일이 캔버스에 잔존
- 사용자가 업로드한 이미지, 생성한 격자(Grid) 등이 캔버스에 표기되지 않음 (썸네일에만 표기됨)
- 편집 완료 후 편집 버튼 재진입 시 초기설정 내용이 초기화됨

**변경 요구사항:**
- 캔버스에서 목업 데이터 설비 및 OHT 레일 완전 제거
- 사용자가 업로드한 배경 이미지, 격자(Grid), 설비 배치 영역, OHT 레일 등이 캔버스에 정상 렌더링되도록 수정
- 편집 완료 후 편집 버튼 재진입 시 기존 초기설정 내용(배경 이미지, 축척, 격자, 영역, OHT 레일 등) 불러와서 수정 가능하도록 구현

---

## 수정 항목 4 — 최적화 실행 버튼 위치 변경

**현재 상태:**
- `최적화 실행` 버튼이 편집 모드에 위치

**변경 요구사항:**
- 편집 모드에서 `최적화 실행` 버튼 제거
- 보기 모드의 기본 기능 패널로 이동
