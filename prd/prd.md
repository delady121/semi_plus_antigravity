# [PRD] Semi-PLUS: 통합 FAB 레이아웃 및 협업 시스템

**프로젝트명:** Semi-PLUS (Semi-Plan & Layout Unified System)  
**버전:** 1.0  
**상태:** 최종 확정 (개발 단계 진입 가능)  
**작성자:** IT 서비스 기획자(PM) & 시스템 설계 파트너

---

## 1. 프로젝트 개요 (Project Overview)

### 1.1 배경 및 목적
* **현상:** 제조 기획 업무 데이터가 개별 Excel 및 Spotfire로 파편화되어 관리됨.
* **문제점:** * 데이터 수정 시 유관 부서 간 동기화 누락 빈번.
  * 기존 사내 시스템은 단순 도면 조회만 가능하여 실제 배치 검토에 한계.
  * 수치 기반의 수동 배정으로 인한 **Dead Space** 발생 및 설비 간 간격(Maintenance) 확보 오류.
* **해결 방안:** 데이터 통합 DB 구축, 상태 기반 협업 워크플로우 내재화, Canvas 기반 지능형 설비 배치 시뮬레이터 개발.

### 1.2 핵심 가치 (Goals)
1. **Single Source of Truth:** 모든 부서가 동일한 최신 데이터를 참조하는 환경 구축.
2. **Space Optimization:** 알고리즘을 통한 데드 스페이스 최소화 및 최적 배치 제안.
3. **Custom Dashboard:** 위젯 기반의 자유로운 화면 구성 및 전역 데이터 분석 환경 제공.

---

## 2. 사용자 시나리오 (User Scenarios)

### 2.1 설비 배치 및 공간 검토 (제조 기획자)
1. **데이터 마스터 구성:** Datalake API 연동 및 전역 테이블 구성(계산된 컬럼 포함).
2. **도면 시뮬레이션:** 600mm 그리드와 OHT 레일이 포함된 도면 위에서 설비를 자유 배치. 드래그 중 실시간 거리(HUD) 확인.
3. **최적화 분석:** 시스템 추천 알고리즘을 통해 반출 일정과 제약 구역을 고려한 최적 후보지 확인.
4. **협업 요청:** 확정된 가안(Draft)을 바탕으로 기술/건설 등 유관 부서에 검토 요청 발송.

### 2.2 협업 및 승인 프로세스 (유관 부서 담당자)
1. **상태 관리:** 요청 수신 후 `처리 중`으로 변경하여 업무 착수 알림.
2. **실시간 수정 대응:** 요청자가 `처리 중` 상태에서 내용을 수정하거나 취소할 경우 즉시 알림 수신.
3. **최종 합의:** 모든 지정 부서가 `완료`를 눌러야 전체 요청 건이 `최종 완료`로 전이됨.

## 3. 핵심 기능 목록 (Key Features)

### 3.1 지능형 레이아웃 시뮬레이터 (Layout Engine)
* **7대 레이어 시스템:**
    1. **IMG:** 고정 시설물 베이스 이미지.
    2. **영역:** 배치 가/불가 구역 (시스템 내 드로잉 툴로 생성).
    3. **OHT:** 설비 전면부 정렬 기준 레일 (시스템 내 생성/스냅 지원).
    4. **Grid:** 600mm 간격 시각적 가이드 (자동 생성).
    5. **설비:** 실시간 Datalake API 연동 설비 객체 (좌표 기반 사각형).
    6. **배치:** 사용자 마킹, 메모, 보관소 표기용 커스텀 레이어.
    7. **검토:** 알고리즘 결과 및 공정별 제약 사항 표기.
* **인터랙션:** 드래그 앤 드롭 배치, **OHT Snap(전면 자동 정렬)**, 실시간 거리 HUD.
* **유지보수 공간(Maintenance):** 설비별 개별 수치 입력 가능(Default 800mm), 드래그 시 인접 설비 간격 실시간 계산.

### 3.2 전역 데이터 엔진 (Global Data Engine)
* **테이블 빌더:** * `Origin_Table`: 엑셀 벌크 업로드(Ctrl+V), 데이터 타입 지정, 직접 수정(W) 가능.
    * `DB_Table`: 기존 테이블 조합 및 Join (Read-Only).
* **GUI 수식 빌더:** `IF/ELSE`, `CASE WHEN`, `AVG_DISTINCT` 등 고급 함수 조합 지원 및 구문 검사.
* **Marking 연동:** 차트(Web-Spotfire) - 테이블 - 도면 간 상호 데이터 하이라이트/필터링.

## 4. 화면 구성 (Screen Composition)

### 4.1 관리자 네비게이션 및 위젯 설정
* 대분류/소제목 구성을 통한 메뉴 트리 관리 기능.
* 그리드 바탕 위에 테이블, 시각화 차트, 레이아웃 위젯을 자유롭게 배치하는 위젯 편집 모드.

### 4.2 레이아웃 시뮬레이터 메인
* **좌측 패널:** 설비 인벤토리, 모델 검색, 커스텀 설비 박스 생성.
* **중앙 영역:** 메인 캔버스 (7대 레이어 토글, 자유 배치 인터랙션).
* **우측 패널:** 선택 설비 상세 속성(좌표, 투자 구분, Maintenance 설정), 최적 배치 추천 리스트.

---

## 5. 상세 기능 명세 (Functional Specifications)

* **OHT Snap 로직:** 설비 전면부가 레일 근접(약 300mm 이내) 시 자동으로 정렬 및 부착.
* **버전 관리 (Dual-Layer):** 3년치 실도면(Layer 1)과 투자 계획안(Layer 2)을 중첩(Overlay)하여 대조 조회.
* **알림 시스템:** `처리 중` 상태에서 수정 발생 시 담당자에게 즉시 푸시 알림 발송.
* **제약 사항 제어:** 배치 불가 영역 진입 시 경고 팝업 및 강행 시 로그 기록.

## 6. 기술 스택 (Tech Stack)

| 구분 | 선정 기술 | 비고 |
| :--- | :--- | :--- |
| **Frontend** | React.js | 컴포넌트 기반 UI 및 대시보드 관리 |
| **Drawing** | Konva.js | Canvas 기반 멀티 레이어 및 고성능 객체 제어 |
| **Charts** | Apache ECharts | Marking 및 연동 필터링 구현 유리 |
| **Backend** | Node.js (Express) | Datalake API 연동 및 데이터 처리 엔진 |
| **Database** | SQLite / BigQuery | 개발(Local) 및 사내 운영 환경 이원화 |
