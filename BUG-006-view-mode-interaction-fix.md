## 버그 1 — 클릭 반응 레이어 설정이 편집 모드에 위치

**현재 상태:**
- FEAT-002에서 구현한 클릭 반응 레이어 설정(라디오 버튼)이
  편집 모드에 들어가 있음
- 보기 모드에서는 해당 기능이 보이지 않음

**변경 요구사항:**
- 클릭 반응 레이어 설정을 편집 모드에서 제거
- 보기 모드 도구 모음 또는 우측 패널에 배치
- 보기 모드에서의 마우스 인터랙션 정확한 동작:

  **좌클릭 — 개체 선택**
  - 클릭 반응 설정에서 선택된 레이어의 개체만 선택 가능
  - 개체 클릭 시 해당 개체 선택 및 강조 표시

  **좌클릭 드래그 — 범위 다중 선택**
  - 빈 공간에서 좌클릭 드래그 시 선택 범위 사각형 생성
  - 사각형 범위 안에 포함된 개체 다중 선택
  - 선택 범위 사각형은 반투명 파란색으로 표시

  **우클릭 드래그 — 도면 이동(패닝)**
  - 우클릭 드래그 시 도면 전체 이동
  - 드래그 중 커서 모양 변경 (grab → grabbing)

- 클릭 반응 설정 라디오 버튼 목록:
  - 설비 (기본값)
  - 시설물
  - 배치 영역
  - OHT 레일
  - 사용자 정의 레이어 (생성된 경우 목록에 추가)

## 버그 2 — 배치 영역 / OHT 레일 데이터가 데이터 관리에 미표시

**현재 상태:**
- 레이아웃 편집에서 생성된 배치 영역 및 OHT 레일이
  데이터 관리 메뉴에 테이블로 표시되지 않음

**원인 추정:**
- BUG-005 수정 2번(③)에서 데이터 관리 테이블 자동 생성 로직이
  실제로 데이터 관리 페이지의 테이블 목록과 연동되지 않았을 가능성
- layoutEditorStore와 DataManagementPage가
  동일한 데이터 소스를 참조하지 않을 가능성

**변경 요구사항:**
- 레이아웃 편집에서 배치 영역 또는 OHT 레일 생성 시
  데이터 관리 메뉴에 아래 테이블이 자동으로 생성 및 표시
  - `{레이아웃명}_배치영역`
  - `{레이아웃명}_OHT레일`
- 데이터 관리 페이지 진입 시 해당 테이블이 목록에 정상 표시
- 테이블 클릭 시 생성된 배치 영역 / OHT 레일 데이터 조회 가능
- 사내망 이관 시 교체될 수 있도록 데이터 저장부에 주석 표기:
```typescript
// [사내망 이관 시 교체] 데이터 관리 테이블 저장 실제 API로 교체 필요
```
