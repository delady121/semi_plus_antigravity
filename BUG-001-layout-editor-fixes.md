## 버그 1 — 배치 영역 지정 도구 미작동

**현재 상태:**
- 안내 문구에 "도구 모음에서 배치영역 선택 후 캔버스에서 드래그하세요" 라고 표기됨
- 실제 도구 모음에 배치 영역 버튼이 존재하지 않음
- 유사한 사각형 버튼 클릭 시 아무 반응 없음
- 캔버스 자체는 정상적으로 보이는 상태

**변경 요구사항:**
- 도구 모음에 `배치 영역` 버튼 추가
- 버튼 클릭 후 캔버스에서 드래그 시 사각형 배치 가능 영역이 생성되어야 함
- 생성된 영역은 설비 배치 가능 구역으로 시각적으로 구분 표기

---

## 버그 2 — OHT 레일 그리기 도구 미작동

**현재 상태:**
- OHT 레일을 그릴 수 있는 도구가 작동하지 않음
- 캔버스 자체는 정상적으로 보이는 상태

**변경 요구사항:**
- 도구 모음에 `OHT 레일` 그리기 버튼 추가
- 버튼 클릭 후 캔버스에서 클릭하여 선을 그릴 수 있어야 함
- 선 그리기 시 **45도 단위 스냅핑** 적용 (0°, 45°, 90°, 135°, 180°)

---

## 버그 3 — 설비 레이어 설정: 테이블 선택 방식 변경

**현재 상태:**
- 테이블 선택 UI가 드롭다운 검색 기능 없이 구현됨

**변경 요구사항:**
- 업무 시스템 내 데이터 테이블 선택 시 **검색 기능이 있는 드롭다운** 방식으로 변경
- 검색어 입력 시 테이블 목록이 실시간으로 필터링되어야 함

---

## 버그 4 — 설비 레이어 설정: 컬럼 매칭 방식 변경

**현재 상태:**
- 각 컬럼(Xmax, Xmin, Ymax, Ymin, EQP_ID) 매칭이 드롭다운 방식이 아님

**변경 요구사항:**
- 각 필수 컬럼 매칭을 **드롭다운 선택 방식**으로 변경
- 대상 컬럼 목록: Xmax / Xmin / Ymax / Ymin / EQP_ID (필수 5개)
- 추가 컬럼 지정도 동일하게 드롭다운 방식 적용

---

## 버그 5 — 보기 모드에서 도면 백지 상태

**현재 상태:**
- 초기 레이아웃 설정 완료 후 보기 모드 진입 시 백지 상태로 표기됨
- 편집 모드에서는 배경 이미지와 격자가 정상적으로 보임

**원인 추정:**
- 편집 모드에서 렌더링되는 Canvas 데이터가 보기 모드로 전달되지 않음
- 보기 모드가 layoutEditorStore의 상태를 참조하지 않거나
  별도의 데이터 소스를 바라보고 있을 가능성

**변경 요구사항:**
- 보기 모드에서도 편집 모드와 동일하게 배경 이미지, 격자, 배치 영역, OHT 레일 등 정상 렌더링
- 편집 모드와 보기 모드가 동일한 저장 데이터를 참조하도록 수정

---

## 버그 6 — 편집 모드에서 도면 영역 크기 불일치

**현재 상태:**
- 좌측 레이어 탭 선택 시: 도면 영역이 위아래로 협소하게 표시됨
- 설비 탭 선택 시: 도면 영역이 위아래로 가득 차게 표시됨
- 탭에 따라 도면 영역 크기가 달라지는 레이아웃 불안정 상태

**변경 요구사항:**
- 어떤 탭을 선택하더라도 도면 영역 크기가 일정하게 유지되어야 함
- 도면 영역은 탭 선택과 무관하게 항상 동일한 크기로 렌더링

---

## 버그 7 — 편집 모드 재진입 시 초기설정 수정 불가

**현재 상태:**
- 초기 레이아웃 설정 완료 후 편집 모드 재진입 시
  기존에 설정한 내용(배경 이미지, 축척, 격자, 배치 영역, OHT 레일 등)을 수정할 수 없음

**변경 요구사항:**
- 편집 모드 재진입 시 초기설정 항목들을 언제든지 수정 가능하도록 구현
- 수정 가능한 항목:
  - 배경 이미지 재업로드
  - 축척 재설정
  - 격자 설정 수정
  - 배치 영역 추가 / 수정 / 삭제
  - OHT 레일 추가 / 수정 / 삭제
  - 설비 레이어 컬럼 매칭 수정
  - 시설물 레이어 컬럼 매칭 수정
  - 사용자 정의 레이어 추가 / 수정 / 삭제
